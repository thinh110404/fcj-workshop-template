### Week 6 Objectives:

- Understand the AWS Shared Responsibility Model.
- Learn the basic security concepts in AWS account and identity management.
- Study AWS IAM Root Account, AWS Organizations, IAM Admin User, and AWS Identity Center.
- Learn AWS Key Management Service (KMS) and basic encryption concepts.
- Practice configuring IAM Policies and Multi-Factor Authentication (MFA).
- Join the event

### Week 6 Achievements:

- Understood the AWS Shared Responsibility Model and distinguished the security responsibilities between AWS and the customer.

- Recognized the importance of securing the AWS account, especially the root account.

- Understood why root account credentials should be protected and why the root account should not be used for daily operations.

- Reviewed the knowledge of AWS Organizations and IAM Admin User in managing accounts and access permissions.

- Understood the basic role of AWS Identity Center and Single Sign-On in centralized access management.

- Understood the role of AWS Key Management Service (KMS) in encryption and data protection.

- Practiced configuring IAM Policy and checking access permissions.

- Practiced setting up Multi-Factor Authentication (MFA) to strengthen user security.

- Improved awareness of security governance and best practices when working with AWS services.
